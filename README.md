# TextStar Landing Page

This repository contains the public landing page for TextStar, a distraction-free writing editor for serious writers.

TextStar is designed around a simple idea:

The words come first.

The landing page introduces the product philosophy, explains who TextStar is for, and presents the app as a calm, focused writing environment for authors, essayists, and long-form writers.

**Landing Page URL**: [GetTextStar.com](https://gettextstar.com)

## About TextStar

TextStar is a distraction-free editor designed for serious writing.

Whether you are drafting a novel, shaping an essay, or capturing a quiet idea, TextStar gives you a clean, focused space where the words come first.

It is built for writers who want fewer interruptions, less clutter, and a writing tool that stays out of the way.

## Landing Page Goals

This site is meant to communicate that TextStar is:

* Quiet
* Minimal
* Writer-first
* Keyboard-friendly
* Distraction-free
* Designed for long writing sessions
* Coming to the Mac App Store

TextStar is not positioned as an office suite, a cloud collaboration platform, or an AI writing assistant. It is a focused place to write.

## Key Messages

The landing page emphasizes:

* Full-screen distraction-free writing
* A clean interface with no unnecessary visual noise
* Keyboard commands that help writers keep their hands on the keyboard
* No spell-checker
* No AI
* No grammar suggestions
* Plain text friendly writing
* EPUB export
* Mac-first availability

## Development

This repo is intended to be simple, static, and easy to deploy and maintain.

Current structure:
```
/
  index.html
  support.html
  contact.html
  styles.css
  assets/
    textstar-hero.jpg
```

Keeping the implementation lightweight. Avoiding unnecessary dependencies, trackers, analytics, or heavy client-side behavior.

## Design Direction

The visual style should feel:

* Calm
* Premium
* Minimal
* Mac-native
* Dark and focused
* Spacious
* Easy to read

The page should feel more like a quiet writing room than a software dashboard.

## Copy Tone

Using restrained, confident language.

Avoiding hype, startup clichés, productivity jargon, and AI-oriented messaging.

Fav phrases:

* “The words come first.”
* “A quiet editor for serious writing.”
* “No spell-checker. No AI. No noise.”
* “The more you write, the less you reach for the mouse.”
